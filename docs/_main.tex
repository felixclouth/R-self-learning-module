\section{What is R?}\label{what-is-r}

R is a programming language and computational environment that is mainly used as ready-to-use software for statistical data analysis, both simple and advanced. Reasons for using R include:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\item
  It is free. R and all its packages (software that extends R's functionality to enable complex analyses and extended functionality) are absolutely free.
\item
  It's open. Science can only progress if there is complete openness about method. In SPSS and other commercial packages, it is not possible to find out exactly what the program is doing. Companies keep this a secret. All R's methods are on public display.
\item
  R is versatile. Statisticians who develop methods used by social scientists usually implement them in R as packages. Through packages, R's functionality extends far beyond many other software packages.
\item
  R is widely used. This means that colleagues in both Academia and Industry will be easier to collaborate with if you understand R. It also mean that there are many resources available to consult to help you use R.
\item
  R is easy*. While R may initially seem difficult -- and there is a steep initial learning curve -- once you have completed these modules you will have passed most of this learning curve and learning more advanced topics in R will come relatively easily. Furthermore, compared to other programming languages like java and C++, R reads a lot more like English and will be easy to understand. Throughout these modules, you will make use of R in combination with Rstudio. While R is the programming language that does all the work of running the computations, Rstudio provides you with an easy-to-use interface that makes working in R more comfortable and easier to understand.
\end{enumerate}

\section{Data as Objects}\label{data-as-objects}

In R, we spend most of our time working with objects. Objects can be almost anything, from numbers like the \texttt{5} you worked with earlier, to whole datasets, graphic figures, and even code itself can be stored and passed around as an object. To create your own object you make use of \texttt{\textless{}-} as you did previously:

\pandocbounded{\includegraphics[keepaspectratio]{_main_files/images/ObjectAssignment.png}}

\ul{As a quick note}: we advise you to code along with these examples so you understand how to implement them in RStudio.

One major type of object you will frequently encounter (and in fact you just created one) is Data. Data is a broad category comprising anything from a single number to a multi-dimensional array, but for now let's focus on vectors.

\subsection{Vectors}\label{vectors}

A vector is \emph{one or more of the same type of things, one after the other}

The function \texttt{c()} can create vectors:

\end{Shaded}

\end{verbatim}

\begin{Shaded}
\begin{Highlighting}[]

\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] "Hagrid" "Harry"
\end{verbatim}

Notice I make vectors of numbers or sentences but not both at the same time - hence ``the same \ul{type} of thing''. Below are examples of different ``single'' objects that can occur together in a vector.

\subsection{Variables}\label{variables}

A variable is a placeholder name that refers to some object, such as a vector. They function like the numbers they have on clothing racks in some places:

\begin{itemize}
\item
  The coat is the value and the number on the hanger is the name of the variable
\item
  So, \texttt{c(22,\ 22,\ 21,\ 25,\ 23,\ 19,\ 23,\ 25,\ 19,\ 21,\ 20,\ 23)} is the coat, and \texttt{ages} is the ``hanger''.
\end{itemize}

Putting the coat on a hanger is called ``assigning'' a value to a variable.~ As mentioned before assigning value into variables is done with \texttt{\textless{}-}.

A variable can be named anything as long as it does \ul{not} start with a number, does not contain operators, contains at least one regular letter character, and is preferably not already in use by R.

\texttt{rnorm} is a function that \ul{takes} an integer (input) and \ul{returns} that number of random observations from a standard normal distribution (output).

The input of a function is called its \ul{arguments}. Looking at our previous examples:

\begin{itemize}
\item
  \texttt{ages} is the argument in \texttt{sum(ages)}
\item
  \texttt{10} is the argument in \texttt{rnorm(10)}
\end{itemize}

Functions can have multiple arguments:

\begin{Shaded}
\begin{Highlighting}[]

Arguments can be specified either by name, or position:

\begin{itemize}
\tightlist
\item
  \texttt{rnorm(10,\ mean\ =\ 4,\ sd\ =\ 2)} does the same as \texttt{rnorm(10,\ 4,\ 2).}
\end{itemize}

\pandocbounded{\includegraphics[keepaspectratio]{_main_files/images/HelpFiles2.png}}

We can see that for \texttt{rnorm}, the mean and standard deviation of the normal distribution from which \texttt{n} observations are drawn is \texttt{0} and \texttt{1} respectively. This means that if we don't specify \texttt{mean} and \texttt{sd}, and run \texttt{rnorm(10)}, we will draw 10 observations from a normal distribution with a mean of 0 and a standard deviation of 1.

We can also see that \texttt{n} does not have a default argument, which means that when we do not specify it, R gives us the following error:

\begin{Shaded}

To resolve an error/warning first carefully read it.

\begin{Shaded}
\begin{Highlighting}[]
\FunctionTok{sum}\NormalTok{(}\FunctionTok{c}\NormalTok{(}\StringTok{"Happy Meal"}\NormalTok{, }\StringTok{"Roger Rabbit"}\NormalTok{))}

\subsection{Indexing}\label{indexing}

\ul{Indexing} in R refers to selecting a subset of elements from an object. Returning to our \texttt{ages} example:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{ages }\OtherTok{\textless{}{-}} \FunctionTok{c}\NormalTok{(}\DecValTok{22}\NormalTok{, }\DecValTok{22}\NormalTok{, }\DecValTok{21}\NormalTok{, }\DecValTok{25}\NormalTok{, }\DecValTok{23}\NormalTok{, }\DecValTok{19}\NormalTok{, }\DecValTok{23}\NormalTok{, }\DecValTok{25}\NormalTok{, }\DecValTok{19}\NormalTok{, }\DecValTok{21}\NormalTok{, }\DecValTok{20}\NormalTok{, }\DecValTok{23}\NormalTok{)}

\end{Shaded}

In that case, you can use the following code:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{ages[}\DecValTok{4}\NormalTok{]}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 25
\end{verbatim}

\subsubsection{Indexing multiple elements}\label{indexing-multiple-elements}

You index an element from a vector in R by placing the location of the element you want to index between square brackets \texttt{{[}\textasciigrave{}\textasciigrave{}…\textasciigrave{}\textasciigrave{}{]}} after the vector.

The \texttt{{[}…{]}} can also take vectors, in case you want more than one element from the vector:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{ages[}\FunctionTok{c}\NormalTok{(}\DecValTok{1}\NormalTok{, }\DecValTok{3}\NormalTok{, }\DecValTok{4}\NormalTok{)]}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 22 21 25
\end{verbatim}

In case you want the second through fifth elements of a vector:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{ages[}\FunctionTok{c}\NormalTok{(}\DecValTok{2}\NormalTok{, }\DecValTok{3}\NormalTok{, }\DecValTok{4}\NormalTok{, }\DecValTok{5}\NormalTok{)]}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 22 21 25 23
\end{verbatim}

It can be easy to use the \texttt{:} operator.

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{ages[}\DecValTok{2}\SpecialCharTok{:}\DecValTok{5}\NormalTok{]}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 22 21 25 23
\end{verbatim}

The \texttt{:} operator creates a vector running from the integer at the left side of the \texttt{:} , to the integer at the right side, in steps of 1:

\begin{Shaded}
\begin{Highlighting}[]
\DecValTok{2}\SpecialCharTok{:}\DecValTok{5}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1] 2 3 4 5
\end{verbatim}

\begin{Shaded}

\NormalTok{ages[}\SpecialCharTok{{-}}\DecValTok{4}\NormalTok{]}

\end{Shaded}

\end{verbatim}

\subsubsection{R Packages \& Package Datasets}\label{r-packages-package-datasets}

Sometimes R packages also come with their own built-in datasets. These are loaded in the same way as the built-in R datasets. To access these datasets you do need to have the respective package installed and activated.

R packages are extensions to Rs functionality created by members of the R community. Most of the R packages you will work with are hosted on the Comprehensive R Archive Network (\ul{CRAN}). Packages hosted on this archive can be installed using the following format of code:

\texttt{install.packages("package\_name")} You will only need to install your package onto your system once.

However, each time you start an R session, you will need to activate the package using the following code:

\texttt{library(package\_name)}

\begin{Shaded}
\begin{Highlighting}[]
\FunctionTok{install.packages}\NormalTok{(}\StringTok{"dplyr"}\NormalTok{) }\CommentTok{\#only run this once}

\FunctionTok{library}\NormalTok{(dplyr) }\CommentTok{\#run this everytime you start an R session}
\end{Highlighting}
\end{Shaded}

It is generally advisable to run your \texttt{install.packages()} functions in the console and not store them in your script.

It is good coding standards to put ALL your library functions at the top of your R script.

Sometimes you will get the following pop-up when installing a package:

\pandocbounded{\includegraphics[keepaspectratio]{_main_files/images/InstallingPackage1.png}}

This means that there has been a recent update that is not available for your operating system yet. You have two options:

\begin{itemize}
\item
  ``Yes'': build the latest version from source binary. Takes longer, you need extra packages, but you'll have the latest version.
\item
  ``No'': build the previous version from binary files. A lot faster, no extra packages needed, but you won't have the latest version (often not a problem).
\end{itemize}

\subsubsection{Conditional Selection}\label{conditional-selection}

So far, when we have indexed datasets or vectors, we have made use of names (e.g.,~~ \texttt{llama\_age{[}‘Zaphod’{]}}, or \texttt{StarWars\$height)}, or using numeric values to indicate locations (e.g., \texttt{ages{[}c(1,\ 3,\ 4){]}}). We can also use vectors of logical (TRUE/FALSE) values for indexing.

To index with a vector of logical values the vector must be the same length as the object you want to index over.

Let's say for example, we wanted to use logical values to index over the \texttt{sleep} dataset, which has 20 rows and 3 columns.

\section{Recoding}\label{recoding}

In Module 2 we said that you should always check your data as the first step in your analyses. After checking your data, your second step will often be to clean your data. Raw data is often messy and requires you to treat missing values and errors in the data.

In some statistical analysis programs, you as a user specify what is a missing value is. These are often improbable values like 99.

\pandocbounded{\includegraphics[keepaspectratio]{_main_files/images/SPSSMissingDataExample.png}}

R will treat these as numbers (because they are), not as missings. We need to \ul{recode} these values to NAs.

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{vector }\OtherTok{\textless{}{-}} \FunctionTok{c}\NormalTok{(}\DecValTok{4}\NormalTok{, }\DecValTok{1}\NormalTok{, }\DecValTok{6}\NormalTok{, }\DecValTok{99}\NormalTok{, }\DecValTok{9}\NormalTok{)}

\NormalTok{vector}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}

\end{verbatim}

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{vector }\OtherTok{\textless{}{-}} \FunctionTok{c}\NormalTok{(}\DecValTok{4}\NormalTok{, }\DecValTok{1}\NormalTok{, }\DecValTok{6}\NormalTok{, }\DecValTok{99}\NormalTok{, }\DecValTok{9}\NormalTok{)}
\NormalTok{vector}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}

\end{verbatim}

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{vector[vector }\SpecialCharTok{==} \DecValTok{99}\NormalTok{] }\OtherTok{\textless{}{-}} \ConstantTok{NA}
\NormalTok{vector}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
## [1]  4  1  6 NA  9
\end{verbatim}

\section{Doing Something to every row/column}\label{doing-something-to-every-rowcolumn}

One of the advantages of R we have already discussed is that it is a vectorized programming language. Meaning, that in most cases if an operation is performed on a vector it is performed on each element in that vector automatically, without the need for us to code our operation to work specifically on a vector.

Things get a little more complicated if we want to perform actions across every row and column in a data frame or matrix. This is not done automatically in R as for vectors. Luckily there are certain functions we can use that do allow us to perform operations across either rows or columns.

\section{Formula Notation}\label{formula-notation}

Most statistical functions in R require you to specify your model with formula notation, in the format: \texttt{y\ \textasciitilde{}\ x}.

\pandocbounded{\includegraphics[keepaspectratio]{_main_files/images/FormulaNotation1.png}}

The analysis function will then take this formula notation and run your model to analyze the relationship between these variables and return the resulting output in a list from which you can then index the relevant results to present (described in more detail later).

If you wanted to add multiple variables, you would do so in the following format:

\texttt{y\ \textasciitilde{}\ x1\ +\ x2\ +\ x3,\ etc.}

If you wanted to add an interaction you would do so as follows:

\texttt{y\ \textasciitilde{}\ x1:x2}

Or equivalently:

\texttt{y\ \textasciitilde{}\ x1\ +\ x2\ +\ x1*x2}

As an overview, here is a table containing various common ways of specifying different formula notations.

\begin{table}

\caption{\label{tab:formula-notation}Formula Notation in R}
\centering
\begin{tabular}[t]{ll}
\toprule
Formula.in.R & Meaning\\
\midrule
y \textasciitilde{} 0 + x /n y \textasciitilde{} x - 1 & No intercept\\
y \textasciitilde{} x + z & Main effects\\
y \textasciitilde{}x:z & Interaction\\
y \textasciitilde{} x*z & Main \& interaction effects /n Equivalent to: /n y \textasciitilde{} x + z + x:z\\
y \textasciitilde{} poly(x, 2) & Quadratic (2nd order polynomial)\\
\addlinespace
y \textasciitilde{} A & ANOVA, A is a factor variable\\
y \textasciitilde{} A + x & ANOVA + covariate\\
y \textasciitilde{} A*B*C - A:B:C & No 3rd order interaction\\
\bottomrule
\end{tabular}
\end{table}

\subsection{Linear Regression}\label{linear-regression}

The function used to conduct a linear regression in r is the \texttt{lm()} function. Typing \texttt{?lm} will show us all the arguments we can specify.

\pandocbounded{\includegraphics[keepaspectratio]{_main_files/images/lmHelpFiles.png}}

There are many potential arguments to specify. Luckily, we can leave most of these functions with the default. We should make sure to specify the following arguments: \texttt{formula} \& \texttt{data}.

As an example, we will use the \texttt{iris} dataset, and try to regress petal width on petal length. We can do this as follows:

\texttt{lm(Petal.Length\ \textasciitilde{}\ Petal.Width,\ data\ =\ iris)}

or

\texttt{lm(iris\$Petal.Length\ \textasciitilde{}\ iris\$Petal.Width)}

\subsection{Logistic Regression}\label{logistic-regression}

A logistic regression is like a linear regression, only used to predict a binary variable (Yes/No, employed/unemployed, etc.) instead of a continuous variable (Sepal.Length, conscientiousness, etc.)

To perform a logistic regression, we make use of the \texttt{glm()} function. This function is a more general form of the \texttt{lm()} function, which can handle outcome variables that are not continuous with normally distributed errors.

However, to use this function properly, we will have to specify an additional argument, the \texttt{family} argument.

Without getting into detail about why and how we must change our family specification when we want to conduct a logistic regression. For now, simply know that you should make sure to specify \texttt{family\ =} \texttt{binomial()}, when you want to conduct logistic regression using \texttt{glm()}.
